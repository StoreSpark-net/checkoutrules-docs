## Rule overview

This rule applies shipping discounts at checkout based on your conditions. In the current app flow, shipping discounts are configured from **Discount all** by enabling **Shipping discount** (you can still manage existing shipping-discount rules created earlier).

You can create either a discount code or an automatic discount, choose shipping method scope, and optionally control subscription-delivery behavior.

## Supported operations

| Operation | Behavior |
| --- | --- |
| **Percentage** | Applies a percentage discount to selected shipping method(s). |
| **Fixed amount off** | Applies a fixed currency amount discount to selected shipping method(s). |
| **Set fixed price** | Sets each selected shipping method to a target fixed price. |
| **Set all shipping free** | Makes all shipping methods free. |
| **Subtotal percentage** | Discount amount is `order subtotal × percent / 100`, then applied to selected shipping method(s). |
| **Fixed price subtotal percentage** | Sets each selected shipping method price to `order subtotal × percent / 100`. |

## Sample configuration

The following steps show how to configure a shipping discount in the current UI.

### Create a new rule

From your [Shopify admin](/guides/en/get-started/installation#go-to-the-app), open the app. In the sidebar, go to [Rules](/guides/en/get-started/quickstart#create-a-sample-rule) and click _"Create Rule"_.

On the **Select checkout rule type** screen, choose:

- category **Discount**
- checkout rule type **Discount shipping methods**
- click **Select Rule**

<img
  src="/images/essentials/shipping/select-discount-shipping-methods.webp"
  alt="Select Discount Shipping Methods"
/>

Next, you’ll land on the discount rule screen.

### Rule configuration

Choose whether this will be an **Automatic discount** or **Discount code**, then set [status](/guides/en/get-started/concepts#status) to **Testing**, **Active**, or **Deactivated**.

<img
  src="/images/essentials/shipping/rule-status-discount-shipping-methods.webp"
  alt="Rule Status"
/>

Configure [Combinations](/guides/en/get-started/concepts#combinations) based on how this discount should interact with product/order discounts.

<img
  src="/images/essentials/shipping/combinations-disount-shipping-methods.webp"
  alt="Combinations Discount Shipping Methods"
/>

Set the [Strategy](/guides/en/get-started/concepts#strategy). For shipping-only discount behavior, this is **First**.

<img
  src="/images/essentials/shipping/strategy-shipping-discount.webp"
  alt="Strategy Shipping Discount"
/>

Then set your [condition](/guides/en/get-started/concepts#conditions-when) and [action](/guides/en/get-started/concepts#action-what).

> Example: set the rule to **Always**, choose **Percentage**, enter `10`, and use message **10% OFF**.

<img
  src="/images/essentials/shipping/condition-set-discount-shipping-methods.webp"
  alt="Condition Set Discount Shipping Methods"
/>

In the action section, choose your discount operation.

<img
  src="/images/essentials/shipping/action-type-discount-shipping-methods.webp"
  alt="Action Type Discount Shipping"
/>

For method targeting, choose one of:

- **Apply this discount to all shipping methods**
- **Apply this discount to only these shipping methods**

If you target specific shipping methods:

- Use **contains / does not contain / is / is not**
- Matching is case-insensitive
- `*` wildcard is supported (for example: `Standard*`, `*Express*`, `*Pickup`)
- Multiple rows are OR logic (any matching row can apply)

<Warning>
  Shopify may show USPS-prefixed names at checkout. For method matching, remove
  the USPS prefix in your input. Example: use `Ground Advantage` instead of
  `USPS Ground Advantage`.
</Warning>

<img
  src="/images/essentials/shipping/action-shipping-methods-selection-discount-shipping-methods.webp"
  alt="Shipping Method Selection"
/>

You can also configure **Subscription deliveries** behavior:

- **Limit shipping discount to first delivery**
- **Limit shipping discount to first delivery and set recurring delivery free**
- **Limit shipping discount to first delivery and set all recurring deliveries free**

If you choose the limited recurring option, set **Discount limit** for the number of free recurring deliveries.

If your rule uses **Date Time** or **Customer location (using GeoIP)** conditions, configure fallback:

- **Do nothing**: discount action is skipped when time/date or location context cannot be determined.
- **Do this action**: the configured fallback action runs instead.

Finally, configure [Maximum Discount Uses](/guides/en/get-started/concepts#maximum-discount-uses) (for discount codes), then click <b>Save</b>.

<img src="/images/shared/save-rule.webp" alt="Save Rule" />

After testing, activate the rule from **Rules** in the sidebar.

<Warning>
  Rule activation is available on paid plans. If needed, upgrade
  [here](/guides/en/get-started/installation#upgrade-your-plan).

## Limitations

- This rule is incompatible with draft-order-based checkout and apps that use draft-order checkout.
- If checkout URL contains `.../checkouts/do/...`, `do` means draft order and this rule will not run.
