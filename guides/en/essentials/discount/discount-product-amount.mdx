## Rule overview

This guide covers product-level discounts in the current app flow. In the current rule creation UI, product discounts are configured from **Discount all** by enabling **Product discount**.

You can create either discount codes or automatic discounts, choose strategy, and apply one of the supported product discount operations.

## Supported product operations

| Operation | Behavior |
| --- | --- |
| **Percentage** | Applies percentage discount to selected products. |
| **Fixed amount off** | Applies fixed currency discount to selected products. |
| **Set fixed amount** | Sets selected products to a fixed target amount. |
| **Set all products free** | Makes selected products free. |
| **Fixed price subtotal percentage** | Sets selected product price to `order subtotal × percent / 100`. |

## Sample configuration

### Create a new rule

From your [Shopify admin](/guides/en/get-started/installation#go-to-the-app), open the app. Go to [Rules](/guides/en/get-started/quickstart#create-a-sample-rule) and click _"Create Rule"_.

Choose:

- category **Discount**
- checkout rule type **Discount product amount**
- click **Select Rule**

<img
  src="/images/essentials/discount/select-discount-product-amount.webp"
  alt="Select Discount Product Amount"
/>

### Rule configuration

Choose **Automatic discount** or **Discount code**, then set [status](/guides/en/get-started/concepts#status).

<img
  src="/images/essentials/shipping/rule-status-discount-shipping-methods.webp"
  alt="Rule Status"
/>

Configure [Combinations](/guides/en/get-started/concepts#combinations).

<img
  src="/images/essentials/discount/combinations-discount-product-amount.webp"
  alt="Combinations Discount Product"
/>

Configure [Strategy](/guides/en/get-started/concepts#strategy):

- **First**: applies first matching product-discount block
- **All**: applies all matching product-discount blocks
- **Maximum**: applies matching product-discount block with highest reduction

<img
  src="/images/essentials/discount/strategy-discount-product-amount.webp"
  alt="Strategy Discount Product"
/>

Set your [condition](/guides/en/get-started/concepts#conditions-when) and [action](/guides/en/get-started/concepts#action-what).

> Example: set rule to **Always**, choose **Percentage**, enter `12`, set message **12% OFF**, and select a product.

<img
  src="/images/essentials/discount/condition-action-discount-product-amount.webp"
  alt="Condition and Action Discount Product"
/>

In product operations, select one or more products where required.

<img
  src="/images/essentials/discount/action-type-discount-product-amount.webp"
  alt="Action Type Discount Product"
/>

<Warning>
  For **Set all products free**, discount value/message and product selection are
  not shown because the operation makes eligible products free directly.
</Warning>

<img
  src="/images/essentials/discount/product-selection.webp"
  alt="Product Selection"
/>

If your rule uses **Date Time** or **Customer location (using GeoIP)** conditions, configure fallback:

- **Do nothing**: no product discount action runs when time/date or location context cannot be determined.
- **Do this action**: fallback action runs instead.

Set [Maximum Discount Uses](/guides/en/get-started/concepts#maximum-discount-uses) if this is a discount code, then click <b>Save</b>.

<img
  src="/images/essentials/discount/maximum-discount-uses.webp"
  alt="Maximum Discount Uses"
/>

<img src="/images/shared/save-rule.webp" alt="Save Rule" />

## Limitations

- This rule is incompatible with draft-order-based checkout and apps that use draft-order checkout.
- If checkout URL contains `.../checkouts/do/...`, `do` means draft order and this rule will not run.
