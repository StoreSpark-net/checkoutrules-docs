## Rule overview

This guide covers order-level discounts in the current app flow. In the current rule creation UI, order discounts are configured from **Discount all** by enabling **Order discount**.

You can create either discount codes or automatic discounts, choose order discount strategy, and apply one of the supported order operations.

## Supported order operations

| Operation | Behavior |
| --- | --- |
| **Percentage** | Applies percentage discount to order amount. |
| **Fixed amount off** | Applies fixed currency discount to order amount. |
| **Percentage excluding these products** | Applies percentage discount to order amount while excluding selected products. |
| **Percentage for only these products** | Applies percentage discount based only on selected products. |
| **Fixed amount off excluding these products** | Applies fixed amount discount while excluding selected products. |
| **Fixed amount off for only these products** | Applies fixed amount discount based only on selected products. |
| **Fixed price subtotal percentage** | Sets order amount to `order subtotal × percent / 100` (excluding shipping and tax). |

## Sample configuration

### Create a new rule

From your [Shopify admin](/guides/en/get-started/installation#go-to-the-app), open the app. Go to [Rules](/guides/en/get-started/quickstart#create-a-sample-rule) and click _"Create Rule"_.

Choose:

- category **Discount**
- checkout rule type **Discount order amount**
- click **Select Rule**

<img
  src="/images/essentials/discount/select-discount-amount.webp"
  alt="Select Discount Order Amount"
/>

### Rule configuration

Choose **Automatic discount** or **Discount code**, then set [status](/guides/en/get-started/concepts#status).

<img
  src="/images/essentials/shipping/rule-status-discount-shipping-methods.webp"
  alt="Rule Status"
/>

Configure [Combinations](/guides/en/get-started/concepts#combinations).

<img
  src="/images/essentials/discount/combinations-discount-order-amount.webp"
  alt="Combinations Discount Order"
/>

Configure [Strategy](/guides/en/get-started/concepts#strategy):

- **First**: applies the first matching order-discount block
- **Maximum**: applies the matching order-discount block with highest reduction

<img
  src="/images/essentials/discount/strategy-order-discount.webp"
  alt="Strategy Order Discount"
/>

Set your [condition](/guides/en/get-started/concepts#conditions-when) and [action](/guides/en/get-started/concepts#action-what).

> Example: set rule to **Always**, choose **Percentage**, enter `10`, and set message **10% OFF**.

<img
  src="/images/essentials/discount/condition-set-discount-order-amount.webp"
  alt="Condition Set Discount Order"
/>

When using operations that reference **only/excluding selected products**, add the required product selection in the operation.

<img
  src="/images/essentials/discount/action-type-discount-order-amount.webp"
  alt="Action Types Discount Order"
/>

If your rule uses **Date Time** or **Customer location (using GeoIP)** conditions, configure fallback:

- **Do nothing**: no order discount action runs when time/date or location context cannot be determined.
- **Do this action**: fallback action runs instead.

Set [Maximum Discount Uses](/guides/en/get-started/concepts#maximum-discount-uses) if this is a discount code, then click <b>Save</b>.

<img src="/images/shared/save-rule.webp" alt="Save Rule" />

## Limitations

- This rule is incompatible with draft-order-based checkout and apps that use draft-order checkout.
- If checkout URL contains `.../checkouts/do/...`, `do` means draft order and this rule will not run.
