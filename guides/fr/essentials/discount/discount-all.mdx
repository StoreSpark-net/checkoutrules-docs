## Aperçu de la règle

Cette règle vous permet d’appliquer plusieurs types de remises ensemble en fonction des conditions définies. Vous pouvez combiner des remises sur la commande, le produit et la livraison au sein d’une seule règle, selon les critères configurés. Chaque type de remise peut être personnalisé pour appliquer un pourcentage de réduction, un montant fixe ou d’autres formats de remise pris en charge.

La règle évalue les conditions spécifiées, telles que les tags clients, les tags produits, l’adresse de livraison ou toute autre condition configurée, et applique toutes les remises éligibles en conséquence. Cela offre la flexibilité de créer des offres promotionnelles complètes qui affectent différentes parties du paiement simultanément tout en conservant un contrôle total sur la manière dont les remises sont appliquées.

## Exemple de configuration

Les étapes suivantes montrent comment créer et configurer une règle **Remise globale** en utilisant l’application.

### Créer une nouvelle règle

Depuis votre [administration Shopify](/guides/fr/get-started/installation#accéder-à-l’application), ouvrez l’application. Dans la barre latérale, allez dans la section [Règles](/guides/fr/get-started/quickstart#créer-une-règle-d’exemple) et cliquez sur _"Créer une règle"_.

Vous serez ensuite redirigé vers l’écran **Sélectionner le type de règle de paiement**. Choisissez _catégorie "Remise" → sélectionnez le type de règle de paiement "Remise globale" → cliquez sur “Sélectionner la règle”_.

<img
  src="/images/essentials/discount/select-discount-all.webp"
  alt="Sélectionner Remise globale"
/>

Ensuite, vous arriverez sur l’écran **Créer une nouvelle règle de remise**, où vous pourrez configurer votre règle.

### Configuration de la règle

Pour configurer la règle, ajoutez éventuellement un nom tel que "Remise globale" pour décrire son fonctionnement si vous souhaitez appliquer une **"Remise automatique"**.

<img
  src="/images/essentials/discount/automatic-discount-name-discount-all.webp"
  alt="Nom de la remise automatique"
/>

Ou, si vous préférez appliquer la remise avec un code de réduction, cliquez sur **"Code de réduction"** et saisissez le code souhaité ou générez-en un aléatoirement en cliquant sur **"Générer un code aléatoire"**.

<img
  src="/images/essentials/discount/discount-code-discount-all.webp"
  alt="Code de réduction"
/>

Ensuite, sur le même écran, sélectionnez le [statut](/guides/fr/get-started/concepts#statut) de la règle : Test, Actif ou Désactivé. Ici, nous allons choisir le statut **Test**, afin de pouvoir tester rapidement la règle sans affecter le paiement des clients réels.

<img
  src="/images/essentials/shipping/rule-status-discount-shipping-methods.webp"
  alt="Statut de la règle"
/>

Dans le cadre des paramètres de [statut](/guides/fr/get-started/concepts#statut), la section facultative **Planification** vous permet de contrôler quand une règle active démarre et s’arrête. Sélectionnez une **Date de début** et une **Heure de début** pour indiquer quand la règle doit commencer. Cochez éventuellement **Définir une date de fin** pour choisir une date et une heure de fin auxquelles la règle sera automatiquement désactivée.

> Si vous sélectionnez des dates de planification alors que la règle est définie sur **Actif**, le statut devient **Programmée** au lieu d’Actif. La règle s’exécutera de la date et de l’heure de début jusqu’à la date et de l’heure de fin, si une date de fin est définie.

<img
  src="/images/essentials/discount/schedule-reject-discount-codes.webp"
  alt="Planification Rejeter les codes de réduction"
/>

<Note>
  Pour cet exemple, nous n’utilisons pas de planification et conservons le statut
  **Test**.
</Note>

Ensuite, sur le même écran, configurez les paramètres de [Combinaisons](/guides/fr/get-started/concepts#combinaisons) pour contrôler comment cette règle interagit avec d’autres remises. Vous pouvez choisir de la combiner avec :

- **Remises sur les produits** – Cela permet à la règle Remise globale d’être utilisée avec toutes les remises au niveau produit. Assurez-vous que les remises produit sont également configurées pour se combiner avec toutes les remises.

- **Remises sur la commande** – Cela permet à la règle Remise globale de fonctionner avec les remises globales sur la commande. Assurez-vous que les remises sur la commande sont également configurées pour se combiner avec toutes les remises.

- **Remises sur la livraison** – Cela permet à la règle Remise globale d’être utilisée avec les remises sur la livraison. Assurez-vous que les remises sur la livraison sont également configurées pour se combiner avec toutes les remises.

<img
  src="/images/essentials/discount/combinations-discount-order-amount.webp"
  alt="Combinaisons Remise globale"
/>

Ensuite, sur le même écran, configurez les paramètres de [Stratégie](/guides/fr/get-started/concepts#stratégie) pour contrôler comment le système gère plusieurs blocs de remise. Vous pouvez choisir **Premier**, qui applique uniquement la première remise qui répond aux conditions, en ignorant les remises plus avantageuses qui pourraient suivre, ou **Maximum**, qui évalue tous les blocs de remise et applique celui avec la réduction la plus élevée.

<img
  src="/images/essentials/discount/strategy-order-discount.webp"
  alt="Stratégie Remise commande"
/>

Ensuite, sur le même écran, sélectionnez l’[ensemble de conditions](/guides/fr/get-started/concepts#ensemble-de-conditions) pertinent, la [condition](/guides/fr/get-started/concepts#conditions-le-«-quand-») et l’[action](/guides/fr/get-started/concepts#action-le-«-quoi-»). Choisissez la condition appropriée dans le menu déroulant.

> Par exemple, nous choisirons l’**Ensemble de conditions 1** par défaut. Ensuite, nous configurerons la condition de cette règle sur **Toujours**, afin qu’elle s’exécute dans toutes les situations.

<img
  src="/images/essentials/discount/set-condition-set-discount-all.webp"
  alt="Définir la condition"
/>

Après avoir choisi une condition, cochez les cases de remise sur la commande, le produit ou la livraison selon vos besoins. Vous pouvez sélectionner une ou plusieurs cases selon vos besoins.

<img
  src="/images/essentials/discount/action-checkboxes.webp"
  alt="Cases à cocher pour l’action"
/>

Après avoir choisi les cases, vous devez sélectionner l’action pour ce **type de remise**.

**Remise sur la commande :**

Vous pouvez choisir d’appliquer la remise en utilisant **Pourcentage**, **Montant fixe** ou **Pourcentage du sous-total à prix fixe** sans avoir besoin de sélectionner des produits spécifiques. Toutes les autres méthodes **Pourcentage excluant ces produits, Pourcentage pour ces produits uniquement, Montant fixe excluant ces produits et Montant fixe pour ces produits uniquement** nécessitent de sélectionner des produits pour que la remise fonctionne.

<img
  src="/images/essentials/discount/action-type-discount-order-amount.webp"
  alt="Type d’action Remise commande"
/>

Ensuite, saisissez la valeur et le message de remise.

<img
  src="/images/essentials/discount/action-order-discount-operations.webp"
  alt="Opérations Remise commande"
/>

> Par exemple, ici nous configurons la valeur de la remise à **14%** en utilisant l’option **Pourcentage**. Le message de remise **14% OFF** sera affiché aux clients lors du paiement. Cette remise de 14% s’applique à la commande. Vous pouvez voir les paramètres dans l’image ci-dessous.

<img
  src="/images/essentials/discount/action-example-discount-order-amount.webp"
  alt="Exemple Remise sur la commande"
/>

**Remise sur le produit :**

Vous pouvez choisir d’appliquer la remise en utilisant **Pourcentage**, **Montant fixe**, **Définir un montant fixe**, **Offrir tous les produits gratuitement** ou **Pourcentage du sous-total à prix fixe**, selon vos besoins.

<img
  src="/images/essentials/discount/action-type-discount-product-amount.webp"
  alt="Type d’action Remise produit"
/>

Ensuite, saisissez la valeur et le message de remise. Sélectionnez un ou plusieurs produits depuis la popup pour appliquer la remise. Vous devez ajouter au moins un produit pour que la remise prenne effet.

<Warning>
  Veuillez noter que <b>Offrir tous les produits gratuitement</b> n’affiche
  aucune valeur, message ou sélection de produit car tous les produits sont
  gratuits.
</Warning>

<img
  src="/images/essentials/discount/action-product-discount-operations.webp"
  alt="Opérations Remise produit"
/>

> Par exemple, ici nous configurons la valeur de la remise à **15** en utilisant l’option **Montant fixe**. Le message de remise **15$ OFF** sera affiché aux clients lors du paiement. Nous sélectionnons ensuite le produit **"The Collection Snowboard: Liquid"** pour appliquer la remise à ce produit. Vous pouvez voir les paramètres dans l'image ci-dessous.

<img
  src="/images/essentials/discount/action-example-discount-product-amount.webp"
  alt="Exemple Remise produit"
/>

**Remise sur la livraison :**

Vous pouvez choisir d’appliquer la remise en utilisant **Pourcentage**, **Montant fixe**, **Pourcentage du sous-total**, **Définir un prix fixe**, **Offrir toute l’expédition gratuitement** ou **Pourcentage du sous-total à prix fixe**, selon vos besoins.

<img
  src="/images/essentials/shipping/action-type-discount-shipping-methods.webp"
  alt="Type d’action Remise livraison"
/>

Vous pouvez également choisir plusieurs options pour appliquer la remise. Les options disponibles sont :

- **Méthodes d’expédition sélectionnées**  
  Appliquez la remise uniquement à des méthodes d’expédition spécifiques en saisissant les **noms** des méthodes d’expédition à inclure.

  <img
    src="/images/essentials/shipping/apply-discount-to-selected-shipping-methods.webp"
    alt="Méthodes d’expédition sélectionnées"
  />

- **Toutes les méthodes d’expédition**  
  Appliquez la remise à toutes les méthodes d’expédition disponibles.

  <img
    src="/images/essentials/shipping/apply-discount-to-all-shipping-methods.webp"
    alt="Toutes les méthodes d’expédition"
  />

- **Méthode d’expédition au prix le plus élevé**  
  Appliquez la remise à la méthode d’expédition dont le prix est le plus élevé.

  <img
    src="/images/essentials/shipping/apply-discount-to-highest-price-shipping-method.webp"
    alt="Méthode d’expédition au prix le plus élevé"
  />

- **Méthode d’expédition au prix le plus bas**  
  Appliquez la remise à la méthode d’expédition dont le prix est le plus bas.

  <img
    src="/images/essentials/shipping/apply-discount-to-lowest-price-shipping-method.webp"
    alt="Méthode d’expédition au prix le plus bas"
  />

- **Méthode d’expédition avec prix**  
  Appliquez la remise en fonction d’un montant saisi, avec la possibilité de cibler les méthodes d’expédition dont le prix est supérieur ou inférieur à la valeur indiquée.

  <img
    src="/images/essentials/shipping/apply-discount-to-shipping-method-with-price.webp"
    alt="Méthode d’expédition avec prix"
  />

> Par exemple, ici nous configurons la valeur de la remise à **16%** en utilisant l’option **Pourcentage**. Le message de remise **16% OFF** sera affiché aux clients lors du paiement. Nous avons également défini **Toutes les méthodes d’expédition** pour appliquer la remise à toutes les méthodes d’expédition. Cette remise de 16% s’applique aux méthodes d’expédition. Vous pouvez voir les paramètres dans l’image ci-dessous.

<img
  src="/images/essentials/discount/action-example-discount-shipping-methods.webp"
  alt="Exemple Remise livraison"
/>

**Rejeter les codes de réduction :**

Sélectionnez un ou plusieurs codes de réduction et saisissez un message de rejet pertinent.

<img
  src="/images/essentials/discount/action-reject-discount-codes.webp"
  alt="Action Rejeter les codes de réduction"
/>

Vous pouvez également ajouter des traductions pour le message de rejet dans plusieurs langues en cliquant sur **Ajouter une traduction**, puis en sélectionnant les langues dans lesquelles vous souhaitez traduire le message. Le message de rejet principal devient le **message par défaut** et s’affiche au passage en caisse lorsqu’aucune traduction ne correspond à la langue du client.

<img
  src="/images/essentials/discount/message-translations-reject-discount-codes.webp"
  alt="Traductions du message Rejeter les codes de réduction"
/>

> Par exemple, nous configurons la règle pour rejeter le code de réduction **"SAVE20"**. Le message de rejet **"Ce code de réduction n’est pas disponible pour le moment"** sera affiché aux clients au passage en caisse lorsqu’ils tentent d’appliquer le code de réduction.

<img
  src="/images/essentials/discount/action-example-reject-discount-codes.webp"
  alt="Exemple Rejeter les codes de réduction"
/>

Ensuite, sur le même écran, si l’option de remise d’expédition est cochée, la section **Livraisons d’abonnement** est également disponible. Pour les produits d’expédition par abonnement, vous pouvez choisir de limiter la remise d’expédition à la première livraison uniquement, de limiter la remise d’expédition à la première livraison et de rendre les livraisons récurrentes gratuites, ou de limiter la remise d’expédition à la première livraison et de rendre toutes les livraisons récurrentes gratuites.

<Warning>
  Les options de la section **Livraisons d’abonnement** s’appliquent uniquement aux
  produits d’expédition qui ont un abonnement. Pour les livraisons de produits
  normaux (sans abonnement), vous n’avez pas besoin de sélectionner d’option. Le
  paramètre par défaut fonctionnera comme prévu.
</Warning>

<img
  src="/images/essentials/shipping/subscription-deliveries.webp"
  alt="Livraisons d’abonnement"
/>

Ensuite, sur le même écran, dans la section [Utilisations maximales de la remise](/guides/fr/get-started/concepts#nombre-maximal-d’utilisations-de-la-réduction), vous pouvez éventuellement activer une limite d’utilisation pour le code de réduction, la limiter à une utilisation par client, ou appliquer les deux si un code de réduction est sélectionné.

<img
  src="/images/essentials/discount/maximum-discount-uses.webp"
  alt="Utilisations maximales de la remise"
/>

Enfin, cliquez sur le bouton <b>Enregistrer</b> en haut pour créer et sauvegarder votre règle.

<img src="/images/shared/save-rule.webp" alt="Enregistrer la règle" />

## Activer la règle

Une fois les tests terminés, nous pouvons activer la règle afin qu’elle commence à apporter des modifications au paiement pour les clients réels. Seules les règles activées peuvent modifier le processus de paiement pour tous les clients.

Cliquez sur **Règles** dans la barre latérale pour afficher toutes les règles. Sélectionnez la règle que vous souhaitez activer, puis cliquez sur le bouton **Activer** pour l’activer.

<Warning>
  L’activation des règles n’est disponible que dans les plans payants. Si vous
  n’avez pas encore mis à jour votre plan, suivez les instructions
  [ici](/guides/fr/get-started/installation#mettre-à-niveau-votre-forfait) pour
  effectuer la mise à niveau.
</Warning>

<img
  src="/images/essentials/discount/activate-the-rule-discount-all.webp"
  alt="Activer la règle"
/>

## Limitations

- Cette règle n’est pas compatible avec les checkouts de commandes provisoires ni avec les applications qui utilisent ce type de checkout.
- Si l’URL de votre checkout contient `…/checkouts/do/…`, `do` signifie « commande provisoire ». Cette règle ne s’appliquera pas à ce checkout.
